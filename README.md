# caderno-tematico-logica-e-poo

🚀 Projeto desenvolvido para o desafio da DIO utilizando engenharia de prompts e IA generativa, com o objetivo de aprofundar os conhecimentos adquiridos no curso e consolidar os estudos em Lógica de Programação e Programação Orientada a Objetos (POO) com Java.

Como estudante de Engenharia de Software e em fase inicial da transição para a área de desenvolvimento, escolhi esse tema por acreditar que ele representa a base fundamental para a construção de soluções de software. Dominar lógica e POO é essencial para evoluir em áreas como desenvolvimento Back-end, estrutura de dados, algoritmos e arquitetura de sistemas.

💡 Além de fortalecer meu aprendizado acadêmico e profissional, este material foi organizado para servir como guia de estudos para outros iniciantes que desejam construir uma base sólida em programação.

Engenharia de Prompts


Prompt 1 - Explicação

Aja como um professor especialista em Java, com experiência em ensinar iniciantes.

Explique detalhadamente o que é Java, sua história, principais características, vantagens, desvantagens, aplicações no mercado e como funciona sua arquitetura.

1. Explicando o que é JAVA e como funciona.
2. 4. Organize os assuntos em ordem crescente de complexidade
4. Com exemplos em Java quando for necessário.Utilizando uma analogia do mundo real.
5. Com detalhes técnicos utilizados no mercado.
6. Evite resumir as explicações, pois estou em processo de aprendizado e preciso que os conteúdos sejam apresentados de forma mais aprofundada.

Dificuldade : Inicialmente, utilizei um prompt solicitando que a IA explicasse o assunto como um especialista. No entanto, a resposta começou diretamente com exemplos e analogias, sem apresentar uma introdução adequada sobre o conceito principal, ou seja, sem explicar primeiro o que é Java, sua finalidade e seu contexto dentro do desenvolvimento de software.
O prompt inicial era muito genérico, o que resultava em respostas superficiais e sem uma sequência lógica de aprendizado. Para melhorar os resultados, passei a solicitar explicações mais detalhadas, começando pelos conceitos fundamentais antes de avançar para exemplos e tópicos mais complexos. Também pedi que os assuntos fossem organizados por ordem crescente de complexidade, tornando o estudo mais estruturado e adequado para iniciantes. Esses ajustes mostraram como a Engenharia de Prompts influencia diretamente a qualidade das respostas obtidas.

Mini guia de estudos - Java 

1. A Plataforma Java
Os Pilares: Diferença entre JVM (Máquina Virtual), JRE (Ambiente de Execução) e JDK (Kit de Desenvolvimento)
.
Portabilidade: O conceito de Bytecode e o princípio "Escreva uma vez, execute em qualquer lugar"
.
Ambiente: Instalação do JDK e configuração de variáveis de ambiente
.
Edições: Diferenças entre Java SE (Standard), Java EE (Enterprise) e Java ME (Micro)
.
2. Sintaxe e Convenções
Estrutura de Classe: Como definir uma classe e o uso obrigatório das chaves { }
.
O Ponto de Entrada: O papel do método public static void main(String[] args)
.
Convenções de Código (Sun): Regras de nomeação para classes (PascalCase), métodos e variáveis (camelCase)
.
Identificadores: Regras para nomes válidos e palavras reservadas
.
Comentários: Uso de comentários de linha (//), bloco (/* */) e Javadocs (/** */)
.
3. Tipos de Dados e Variáveis
Tipos Primitivos: Tipos numéricos (int, double), booleanos (boolean) e caracteres (char)
.
Variáveis de Referência: Como o Java acessa objetos na memória através de endereços
.
Constantes: Uso dos modificadores static e final
.
Classes Wrapper: Conversão de tipos primitivos em objetos e métodos parse
.
4. Operadores e Lógica
Aritméticos: Adição, subtração, multiplicação, divisão e o operador de resto %
.
Atribuição e Incremento: Uso de = e operadores de incremento/decremento (++ e --)
.
Precedência: Ordem em que o Java resolve as operações matemáticas
.
5. Controle de Fluxo e Entrada de Dados
Estruturas de Decisão: Uso de if, else e switch-case
.
Estruturas de Repetição: Implementação de loops com while, do-while e for
.
Captura de Dados: Entrada via teclado usando as classes Scanner ou BufferedReader
.
6. Orientação a Objetos (OO)
Classes e Objetos: Abstração do mundo real para o código e criação de instâncias com new
.
Atributos e Métodos: Definição de estados e ações de um objeto
.
Construtores: Inicialização de objetos e o método especial com o nome da classe
.
Pilares da OO: Estudo de Encapsulamento, Herança (extends), Polimorfismo e Abstração
.
Modificadores de Acesso: Controle de visibilidade com public, private, protected e default
.
7. Manipulação Avançada e Erros
Arrays: Criação e manipulação de vetores de tamanho fixo
.
Manipulação de Texto: Uso da classe String e o método equals() para comparação
.
Tratamento de Exceções: Uso de blocos try-catch para lidar com erros em tempo de execução
.
